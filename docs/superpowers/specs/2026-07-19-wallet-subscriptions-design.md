# 钱包订阅中心改版设计

## 背景与目标

当前钱包页顶部同时展示余额、历史消耗和请求次数，信息职责与用量仪表盘重复；页面也没有使用 new 系统已经提供的订阅套餐、当前订阅和订阅购买接口。本次改版把钱包页收敛为账户资金入口：只保留余额，并让订阅、充值和资金记录各自拥有清晰空间。

## 页面结构

钱包页顶部使用一条紧凑余额卡，展示当前余额与手动刷新。其下使用原生分段切换：

1. **订阅套餐**（默认）：展示当前生效订阅、计费顺序、可购买套餐和购买入口。
2. **余额充值**：保留现有支付方式、金额预设、实付计算与外部浏览器支付。
3. **资金记录**：展示现有充值记录及分页。

删除“历史消耗”和“请求次数”卡片，不在钱包页重复用量仪表盘信息。页面继续使用现有 `AppTheme`、7pt 卡片圆角、紧凑表格和原生 Sheet，不复制 new 系统网页布局。

## 订阅数据与交互

- `GET /api/subscription/plans` 读取启用套餐。
- `GET /api/subscription/self` 读取当前/历史订阅及 `billing_preference`。
- `PUT /api/subscription/self/preference` 修改订阅/钱包计费顺序。
- `POST /api/subscription/{stripe|creem|epay}/pay` 创建购买结算。

当前订阅展示套餐名称、状态、已用/总额度、剩余额度、结束时间和下次重置时间。可购买套餐展示名称、副标题、价格、有效期、额度重置、总额度与限购状态。购买 Sheet 只列出后端已启用且该套餐已配置的支付方式；创建成功后使用现有 `ExternalPaymentLauncher` 交给默认浏览器。

额度统一按 `quotaPerUnit` 换算为美元显示，原始额度不直接冒充 Token。套餐价格按后端 `currency` 显示，不在客户端自行换汇。

## 状态、错误与负载

`WalletManagementStore` 保存余额、充值配置、套餐、订阅和资金记录。首次进入只加载一次；切回页面不重复请求；用户手动刷新时只刷新当前视图所需的数据。资金记录在首次切换到“资金记录”时才读取，翻页只读取记录，不重复请求余额、套餐或充值配置。

没有 Timer、轮询、自动重试或按输入实时请求。失败保留已有数据并显示服务端错误。购买和计费顺序更新期间禁用重复操作。401 继续交给现有集中会话清理；请求日志不得包含订单参数、令牌或支付返回字段。

## 验收

- 钱包页不再出现“历史消耗”和“请求次数”。
- 默认能看到真实套餐和当前订阅；无套餐/订阅时显示诚实空状态。
- 套餐支付方式、价格、限购、有效期、额度和重置周期与接口一致。
- 余额充值和资金记录仍可使用，资金记录按需加载。
- 明暗模式及窄窗口无内容重叠或水平撑破。
- 全量测试、Release 构建、签名、安装和真实界面验证通过后提交并推送当前分支。
