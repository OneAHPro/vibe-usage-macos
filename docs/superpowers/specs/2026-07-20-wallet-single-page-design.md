# 钱包单页布局设计

## 目标

钱包页不再通过分段按钮切换“订阅套餐”和“余额充值”。用户进入页面后即可在同一条纵向浏览路径中看到余额、当前订阅、在线充值和套餐列表；资金记录继续位于下方，不抢占主要操作区域。

## 布局

页面沿用现有 Vibe Usage 的纯色卡片、细边框、绿色金额和紧凑字号，不复制 new 系统网页布局。

1. 顶部余额卡保持不变，刷新按钮更新余额、订阅、套餐和充值配置。
2. “当前订阅”和“在线充值”组成自适应双栏。宽窗口左右并列，窄窗口自动上下排列。
3. “可选套餐”独占一整行，内部套餐继续使用自适应网格。
4. “充值记录”位于页面底部，滚动到该区域时才首次请求数据，并保留分页。

删除“钱包功能”分段选择器及其选择状态。用户无需点击标签才能看到订阅或充值。

## 数据与负载

首次进入仍只调用现有钱包概览、套餐和订阅接口。充值配置已包含在概览请求中，不新增请求。资金记录通过底部视图的生命周期按需加载；没有 Timer、轮询或自动重试。顶部刷新只刷新余额、订阅、套餐和充值配置，不连带刷新未操作的资金记录。

## 验收

- 页面中不存在分段切换器或 `selectedSection`。
- 当前订阅、在线充值和可选套餐同时存在于同一滚动页面。
- 宽窗口当前订阅与在线充值并排，窄窗口自动单列。
- 资金记录无需标签切换，滚动到记录区域后按需加载。
- 全量测试、Release 构建、签名、安装及明暗/宽窄窗口检查通过。
