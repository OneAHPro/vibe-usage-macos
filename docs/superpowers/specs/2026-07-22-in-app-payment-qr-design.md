# 软件内支付二维码设计

## 目标

充值和订阅不再在创建支付订单后自动打开默认浏览器。软件应立即弹出支付窗口，用户使用手机扫描二维码完成支付。

## 支付地址

- Stripe、Creem 和 Waffo 使用后端返回的 HTTPS 结账地址。
- 易支付返回表单 action 和已签名字段。客户端将字段作为 URL query 附加到 action，生成可供手机打开的完整支付地址。
- 仅接受 HTTP/HTTPS 地址，不将链接记录到日志。

## 弹窗

通用 `PaymentQRCodeSheet` 同时服务余额充值和订阅购买。弹窗保持 Vibe Usage 现有纯色卡片、绿色金额和紧凑字号，包含：

1. 支付方式和应付金额。
2. 高清无插值二维码。
3. “复制支付链接”备用操作。
4. “完成支付并刷新”主操作。

不提供默认跳转浏览器按钮。二维码生成失败时显示明确错误，仍允许复制支付链接。

## 数据和负载

弹窗不自动轮询订单状态。用户点击“完成支付并刷新”后，客户端只执行一次钱包概览刷新；如果资金记录曾被加载，再执行一次记录刷新。不使用 Timer、周期任务或失败自动重试。

## 验收

- 余额充值和订阅购买均展示软件内二维码弹窗。
- 主支付路径不再调用 `ExternalPaymentLauncher`。
- 表单支付的所有已签名字段完整进入二维码 URL。
- 弹窗在浅色和深色外观下可读，二维码保持白底黑码。
- 完整测试、Release 构建、签名、安装和真实界面检查通过。
